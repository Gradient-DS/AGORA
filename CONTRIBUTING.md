# Bijdragen aan AGORA

Bedankt voor je interesse in het bijdragen aan AGORA! We waarderen de tijd en moeite die je steekt in het verbeteren van dit project. Dit document bevat richtlijnen om het bijdragen zo soepel mogelijk te laten verlopen.

## Gedragscode

Dit project houdt zich aan een gedragscode. Door deel te nemen, wordt verwacht dat je deze code naleeft. Zie [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) voor meer informatie.

## Hoe kan ik bijdragen?

### Bugs melden

Bugs worden bijgehouden als GitHub Issues. Als je een bug vindt:
1. Zoek eerst in de bestaande issues om te zien of de bug al gemeld is.
2. Als er geen bestaand issue is, maak dan een nieuwe aan.
3. Gebruik een duidelijke en beschrijvende titel.
4. Beschrijf de stappen om de bug te reproduceren.
5. Vermeld de verwachte en werkelijke resultaten.
6. Voeg screenshots of logs toe indien mogelijk.

### Functionaliteit voorstellen

Suggesties voor nieuwe functionaliteiten zijn welkom!
1. Open een issue met het label `enhancement`.
2. Beschrijf duidelijk de voorgestelde functionaliteit en het probleem dat het oplost.
3. Leg uit waarom deze functionaliteit nuttig zou zijn voor andere gebruikers.

### Pull Requests (PR's)

1. Fork de repository en maak een branch aan voor je wijziging (`git checkout -b feature/mijn-nieuwe-feature`).
2. Zorg ervoor dat je code voldoet aan de bestaande stijl en conventies.
3. Schrijf tests voor je wijzigingen indien van toepassing.
4. Zorg dat alle tests slagen.
5. Commit je wijzigingen met duidelijke en beknopte commit-berichten (bij voorkeur in het Nederlands, wees consistent).
6. Push je branch naar je fork (`git push origin feature/mijn-nieuwe-feature`).
7. Open een Pull Request naar de `main` branch van deze repository.
8. Beschrijf je wijzigingen in de PR-beschrijving en link naar gerelateerde issues.
